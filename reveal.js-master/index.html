Asynchronous Programming in JavaScript

In JavaScript, asynchronous programming is a crucial concept that allows your code to run in the background without blocking the execution of other code. Let’s delve into what it means and why it’s essential.

Synchronous Programming

Consider the following synchronous code snippet:

						
							const name = "Ann";
							const greeting = `Hello, my name is ${name}!`;
							console.log(greeting); // "Hello, my name is Ann!"

Even if we call a separate function, like this:

						
							function makeGreeting(name) {
								return `Hello, my name is ${name}!`;
							}
							  
							const name = "Ann";
							const greeting = makeGreeting(name);
							console.log(greeting); // "Hello, my name is Ann!"

The Need for Asynchronous Programming
    Common examples of asynchronous tasks in JavaScript include:
  • Making HTTP requests using fetch()
  • Accessing a user’s camera or microphone using getUserMedia()
  • Asking a user to select files using showOpenFilePicker()
    You can achieve this using
  • Callbacks: Functions passed as arguments to other functions, executed when a task completes.
  • Promises: A cleaner way to handle asynchronous operations, allowing chaining and better error handling.
  • async/await: A modern approach that simplifies working with promises, making code more readable

Using Callbacks

						
							function myDisplayer(some) {
								document.getElementById("demo").innerHTML = some;
							}
							  
							function myCalculator(num1, num2, myCallback) {
								let sum = num1 + num2;
								myCallback(sum);
							}
							  
							myCalculator(5, 5, myDisplayer);

What Are Promises?

    It can be in one of three states:
  • Pending: Initial state, neither fulfilled nor rejected.
  • Fulfilled: Operation completed successfully.
  • Rejected: Operation failed.

Promises can be chained using methods like then(), catch(), and finally()

						
							function fetchData() {
								return new Promise((resolve, reject) => {
								  // Simulate fetching data (e.g., from an API)
								  setTimeout(() => {
									const data = { message: "Data fetched successfully" };
									resolve(data); // Resolve the promise
								  }, 1000);
								});
							}
							  
							// Usage
							fetchData()
								.then((result) => {
								  console.log(result.message);
								})
								.catch((error) => {
								  console.error("Error:", error);
								});

Async/await: A Comfortable Approach to Promises

						
							async function fetchData() {
								return 1;
							}

It ensures that the function always returns a promise.

						
							async function fetchData() {
								return 1;
							  }
							  
							fetchData().then(alert); // Outputs: 1

The await keyword works only inside async functions.

						
							async function fetchWithDelay() {
								let promise = new Promise((resolve, reject) => {
								  setTimeout(() => resolve("done!"), 1000);
								});
							  
								let result = await promise;
								alert(result); // Outputs: "done!"
							}
				
							fetchWithDelay();
